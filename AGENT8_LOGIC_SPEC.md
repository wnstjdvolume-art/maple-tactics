전투는 잘 된다. 이제 "별 합성(승급)"과 "승급 가능 표시(반짝)"가 빠졌으니 아래 로직대로 정확히 구현해라. 수치/규칙은 그대로, 그래픽/배치는 내가 준 캡처 이미지를 참고해라.

# 핵심 데이터
- 별 능력치 배수: 별1 = ×1.0, 별2 = ×1.8, 별3 = ×3.2 (체력·공격력 모두 곱셈)
- 유닛 상태: { key(직업키), star(1/2/3), col, row(보드 위치, 벤치면 없음), items:[] }
- 보유 = 벤치(bench 배열) + 보드(board 배열) 양쪽 모두 포함

# 1. ★별 합성(자동 승급) — 가장 중요, 지금 빠진 것
규칙: **같은 직업 + 같은 별** 유닛이 3개 모이면 자동으로 1개의 "다음 별" 유닛으로 합쳐진다. (벤치/보드 어디에 있든 합산)

```
// 유닛을 영입하거나 이동/생성한 직후 항상 호출
function combineCheck(key, star):
  if star >= 3: return                       // 별3이 최대
  count = countUnits(key, star)              // 벤치+보드에서 (key,star) 개수
  if count >= 3:
    // 3개 제거 (보드에 있던 것의 위치를 기억해서 승급유닛을 그 자리에)
    keepPos = null; removed = 0
    보드에서 (key,star) 유닛을 최대 3개 제거 (첫 번째 제거 시 그 col,row를 keepPos에 저장)
    부족하면 벤치에서 (key,star) 유닛을 제거해 총 3개 채움
    // 승급 유닛 생성
    up = { key, star: star+1, items: 합쳐진 3개의 아이템 합치되 최대3개까지 }
    if keepPos 있고 보드에 빈자리(정원 이내) 있으면: up.col=keepPos.col, up.row=keepPos.row → 보드에
    else: up을 벤치 빈칸에
    이펙트: "⭐ {직업명} {star+1}성 승급!" 텍스트 + 합성 연출
    combineCheck(key, star+1)                 // 연쇄: 별2 3개 모이면 별3까지 자동
```
- 예) 별1 검사 3개 → 별2 검사 1개. 별1 검사 9개 모으면 → 별2 3개 → 별3 1개로 연쇄.
- 능력치는 makeCombatant에서 STAR_MULT[star] 적용(별2=1.8, 별3=3.2).
- countUnits(key,star) = bench와 board를 모두 순회하여 key·star 일치 개수.

# 2. ★승급 가능 "반짝" 표시 — 지금 빠진 것
규칙: **상점 카드**에만 표시한다. "지금 구매하면 합성이 완성되는 경우"에만 금빛 반짝 테두리 + "⭐승급!" 배지를 띄운다. (그냥 2개 들고 있는 것만으로는 표시 안 함)

```
function renderShop():
  // 현재 상점에 떠 있는 같은 직업 카드 수
  shopCount = {}; for card in shop: shopCount[card.key] += 1
  for card in shop(5칸):
    owned = countUnits(card.key, 1)          // 보유한 별1 개수
    ready = (owned + shopCount[card.key] >= 3) // 이 상점의 같은 직업까지 사면 3개 → 승급 완성
    if ready:
      카드에 "금빛 펄스(반짝) 테두리" + 좌상단 "⭐승급!" 배지
    else:
      평범하게
```
- 케이스(모두 ready=금빛):
  - 2개 보유 + 상점에 1개 뜸 → 2+1=3 ✅
  - 0개 보유 + 상점에 같은 직업 3개 뜸 → 0+3=3 ✅
  - 1개 보유 + 상점에 같은 직업 2개 뜸 → 1+2=3 ✅
- "반짝"은 금색 테두리가 밝아졌다 어두워졌다 반복하는 펄스 애니메이션(CSS @keyframes 또는 매 프레임 알파 변동).
- 벤치/보드 유닛에는 이 표시를 하지 않는다(상점에서 구매로 완성될 때만).

# 3. 상점 / 영입
