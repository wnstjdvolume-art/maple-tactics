## 0. 한 줄 컨셉
**"메이플 직업 캐릭터(1~5차전직)를 영입·전직시켜 팀을 짜고, 시너지를 맞춰 몬스터 웨이브를 사냥하는 오토배틀러."**

- 내 유닛 = **직업 캐릭터** (코스트 = **전직 차수 1~5**)
- 적 = **몬스터 웨이브** (라운드 비례 난이도, 5·10·15·20라운드는 **보스전**)
- 장르: 오토배틀러 / 택틱스 · 1인 PvE(라운드 생존) · 세션당 10~15분

> 💡 핵심 아이디어: 메이플의 **전직 시스템(1차→5차)** 이 TFT의 **코스트 1~5** 와 자연스럽게 1:1 매핑된다.

## 1. 핵심 게임 루프 (한 라운드)

```
① 준비 단계
   - 상점에서 직업 캐릭터 영입(골드 소비) → 벤치로
   - 벤치 ↔ 보드 드래그로 배치
   - 같은 직업 3명 = 자동 ⭐전직(승급) (별1→별2→별3)
   - 상점 새로고침(리롤, 2골드), 경험치 구매(4골드)
② 전투 단계 (자동)
   - 내 팀(직업) vs 적 몬스터 웨이브 자동 전투
   - 유닛은 가장 가까운 적에게 이동→공격 (근접/원거리)
   - 시너지 효과 적용
③ 정산
   - 승리: 몬스터 전멸 / 패배: 살아남은 적 수만큼 내 체력 감소
   - 골드: 기본 5 + 이자(10골드당 1, 최대 5) + 연승/연패 보너스 + 승리 보너스
   - 라운드마다 경험치 자동 +2
④ 다음 라운드 → 내 체력 0이면 게임오버 / 20라운드 생존 시 클리어
```

## 5. 경제 / 성장 시스템

- **레벨/경험치**: 레벨 = 보드 정원. 경험치 구매(4골드=+4xp) 또는 라운드 자동 +2.
  - 레벨별 보드 정원: Lv1~9 → 3,3,4,5,6,7,8,9,10
  - 레벨업 필요 경험치: 2,4,8,14,24,40,60,84
- **상점 확률**: 레벨↑ = 고차수(고코스트) 직업 등장 확률↑ (`SHOP_ODDS` 참고).
- **유닛 풀(공유 재고)**: 차수별 보유 수량 제한.
- **골드 이자**: 보유 10골드당 +1(최대 +5) → 굴리기 전략.
- **연승/연패**: 2연속부터 +1, 3연속 +2, 5연속+ +3 보너스 골드.

## 6. 보드 / 전투 규칙
- 전장: **7열 × 6행** 그리드. 아래 3행 = 플레이어, 위 3행 = 적.
- 유닛 AI: 매 틱 가장 가까운 적 탐색 → 사거리 안이면 공격, 아니면 한 칸 이동.
- 근접 사거리 1칸, 원거리 3~4칸.
- 전투 종료: 한쪽 전멸 또는 시간초과(생존자 우세).

## 7. Verse8 바이브 코딩 — 단계별 프롬프트 묶음

> MSU Space에서 Agent 8에게 **순서대로** 입력. 한 번에 다 넣지 말고 단계별 확인.

### 프롬프트 ① — 골격
```
메이플스토리 직업 캐릭터로 만드는 TFT(오토배틀러) 게임을 만들어줘.
- 내 유닛은 메이플 직업(검사/매지션/아처/로그/해적 ...), 적은 메이플 몬스터 웨이브다.
- 화면: 위쪽에 7열x6행 전투 보드, 아래쪽에 벤치(9칸)와 상점(5칸), 우측에 시너지 패널.
- 상단 바: 내 체력(100), 골드, 레벨, 라운드, 연승.
- 흐름: 준비 단계(상점에서 직업 영입→보드 배치) → 전투 시작 → 자동 전투 → 정산 → 다음 라운드.
- 모든 그래픽은 메이플스토리 공식 캐릭터/몬스터/배경 에셋을 사용해줘.
먼저 레이아웃과 라운드 흐름의 뼈대만 만들어줘.
```

### 프롬프트 ② — 직업 유닛과 상점
```
유닛으로 쓸 메이플 직업 21종을 코스트(=전직 차수) 1~5로 추가해줘:
[2번 표 그대로 붙여넣기 — 차수/직업명/계열/역할]
1차전직=코스트1 ... 5차전직=코스트5. 각 유닛은 체력/공격력/공격속도/사거리(근접1, 원거리3~4칸)를 가진다.
상점 5칸은 레벨이 높을수록 고차수 직업이 더 자주 나온다.
카드 클릭=골드 소모해 영입(벤치로). "새로고침"(2골드)으로 상점 리롤.
```

### 프롬프트 ③ — 배치와 전직(승급)
```
벤치의 직업을 드래그해 보드(아래 3행)에 배치. 보드 정원 = 레벨.
같은 직업 3명을 모으면 자동으로 한 단계 전직(⭐2)되어 강해진다(2성 1.8배, 3성 3.2배).
우클릭하면 판매해 골드 환급.
```

### 프롬프트 ④ — 적 몬스터 웨이브 & 자동 전투
```
"전투 시작"을 누르면 자동 전투. 각 유닛은 가장 가까운 적에게 이동, 사거리 안이면 공격속도대로 공격.
적은 메이플 몬스터 웨이브(달팽이~그리피)로, 라운드가 오를수록 수와 능력치가 강해진다.
5/10/15/20 라운드는 보스전: 머쉬맘→자쿰→혼테일→발록(거대 체력+호위 몬스터).
이기면 몬스터 전멸, 지면 살아남은 적 수만큼 내 체력 감소. 데미지 숫자 연출 추가.
```

### 프롬프트 ⑤ — 시너지(트레잇)
```
보드 위 직업들의 시너지를 계산해 우측 패널에 표시. 계열 축과 역할 축 두 가지가 있다.
[3번 표 그대로 붙여넣기 — 계열/역할 시너지와 효과]
시너지는 보드 위 서로 다른 직업 수로 센다. 발동 단계에 도달한 시너지만 효과 적용·강조.
```

### 프롬프트 ⑥ — 경제와 마무리
```
라운드마다 골드 정산: 기본 5 + 이자(보유 10골드당 1, 최대 5) + 연승/연패 보너스 + 승리 +1.
경험치 구매(4골드=+4xp)로 레벨업, 라운드마다 자동 +2. 레벨=보드 정원.
체력 0 = 게임오버, 20라운드 생존 = 클리어. 메이플 공식 BGM/효과음, 결과 팝업 추가.

```
유닛/몬스터/보스를 메이플스토리 공식 캐릭터·몬스터 스프라이트로 교체해줘.
[아래 'MSU 자산 매핑표'의 직업↔공식 캐릭터 / 몬스터↔공식 몬스터 연결을 그대로 사용]

전투 이펙트:

## 10. 차별화 포인트
1. **전직 시스템 = 코스트 시스템** — 메이플 유저에게 직관적인 1~5차 위계.

## 11. 멀티플레이 / PvP 설계 (확장)

> 컨셉 반전: 기존 TFT = **PvP 메인** + 중간 몬스터. 본 게임 = **몬스터(PvE) 메인** + **가끔 라이벌(PvP)**.
> 이유: 동시접속 플레이어가 적을 것을 가정 → PvE를 축으로, 일정 라운드마다 플레이어전이 끼어들고 그 결과가 체력에 반영.

### 라운드 구조 (싱글/멀티 공통 스케줄)
- 일반 라운드: 몬스터 웨이브 (PvE)
- 5·10·15·20R: 보스전 (PvE)
- **4·8·12·16·19R: 라이벌 매치 (PvP)** — 패배 시 살아남은 상대 유닛 수만큼 체력 감소
- 싱글 모드: 라이벌 = AI 생성 팀 / 멀티 모드: 라이벌 = **실제 다른 플레이어 보드 스냅샷**

### ✅ 엔진은 이미 PvP 준비됨 (점검 결과)
- 전투 시뮬(`simStep`/`applyTraitBuffs`/궁극기)은 **팀 vs 팀 대칭** → 적을 "몬스터"에서 "상대 직업 보드"로 **교체만** 하면 됨. (로컬에서 `genRivalTeam()`으로 검증 완료 — 시너지·궁극기·아이템까지 그대로 작동)
- 패배 시 체력 감소 로직 기존 그대로 재사용.

### 멀티 전환 시 서버에 올려야 하는 것 (Verse8/MSU 네트워킹)
| 항목 | 싱글(현재) | 멀티(서버) |
|---|---|---|
| 상대 보드 | AI 생성(`genRivalTeam`) | 다른 플레이어 보드 **스냅샷** fetch |
| 챔피언 풀(`G.pool`) | 로컬 | **공유 풀**(서버 권위) |
| 라운드 타이밍 | 즉시 | 전 플레이어 **동기화**(서버 오케스트레이션) |
| 체력·순위·탈락 | 로컬 | 서버가 각 플레이어 HP/순위 관리 |
| 매칭 | 없음 | 라운드마다 상대 매칭(라운드로빈/랜덤) |

→ **게임 로직은 그대로 포팅**. "상대 보드 가져오기 + 라운드/HP 동기화"만 서버 호출로 바뀜. Verse8 멀티플레이 인프라 활용.

### Verse8 프롬프트(멀티 단계, 나중에)
```
일정 라운드(4·8·12·16)마다 다른 플레이어의 보드와 자동 전투하는 PvP 라운드를 넣어줘.
온라인 매칭으로 상대 보드 스냅샷을 받아 내 보드와 싸우고, 지면 살아남은 상대 유닛 수만큼 체력이 깎인다.
챔피언 풀은 같은 방 플레이어끼리 공유하고, 마지막까지 살아남은 플레이어가 승리.
오프라인/대기 시에는 AI 라이벌로 대체.
```
